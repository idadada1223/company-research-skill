# 就活で使う企業分析フレームワーク

企業研究に使えるフレームワークを、**就活で本当に役立つ使い方**に絞って整理する。
全部使う必要はなく、セクション 4〜6 で使っている PEST・5 Forces・3C・SWOT が押さえられれば十分。

## 1. PEST 分析（マクロ環境）

企業を取り巻く**外部環境**を 4 象限で整理する。業界単位で考えるのが基本。

| 軸 | 観点 | 情報系院生が特に見るべきポイント |
|---|---|---|
| **P**olitics（政治・法規制） | 法改正、規制緩和、税制、補助金 | データ規制（GDPR、改正個人情報保護法）、AI 規制、経済安保 |
| **E**conomy（経済） | 景気、為替、金利、消費動向 | 半導体市況、円安の影響、賃上げトレンド |
| **S**ociety（社会・トレンド） | 人口動態、価値観、ライフスタイル | 少子高齢化、DX 浸透度、ワークスタイル変化 |
| **T**echnology（技術） | 新技術、特許、標準化動向 | 生成 AI、量子、6G、Web3、サイバーセキュリティ |

**使い方**：企業の事業に影響する要因を各象限に 2〜3 個ずつ挙げる。
これが **「業界の将来性」** を語るときの材料になる。

## 2. 5 Forces（業界の競争環境）

Porter の 5 つの力で、**業界の収益性・競争の激しさ**を見る。

1. **既存競合の脅威**：プレイヤー数、上位集中度、製品の同質性
2. **新規参入の脅威**：参入障壁の高さ（資本・技術・規制・ブランド）
3. **代替品の脅威**：他の手段で同じニーズが満たせるか
4. **買い手（顧客）の交渉力**：顧客が価格を押し下げる力
5. **売り手（サプライヤー）の交渉力**：仕入れ先が価格を押し上げる力

**使い方**：「なぜこの業界を選んだか」の根拠として、
「参入障壁が高く」「代替品の脅威が小さく」「利益が出やすい構造」といった視点で語れる。

## 3. 3C 分析（ミクロ環境）

| C | 中身 | 見るべきこと |
|---|---|---|
| **Customer**（市場・顧客） | 誰に売っているか | 顧客層、市場規模、顧客の変化 |
| **Competitor**（競合） | 誰と戦っているか | 主要競合 2〜3 社の強み・弱み |
| **Company**（自社） | 自分たちの強みは何か | 技術、人材、ブランド、顧客基盤 |

**使い方**：志望企業の競争優位を特定する。
「なぜ競合 A 社ではなく、この企業か」の答えがここから生まれる。

## 4. SWOT 分析

3C・PEST・5 Forces の結果を集約する形で作る。

|  | 有利 | 不利 |
|---|---|---|
| **内部** | Strengths（強み） | Weaknesses（弱み） |
| **外部** | Opportunities（機会） | Threats（脅威） |

**必ず順番は「外部（O/T）→ 内部（S/W）」**。
前提となる市場環境を整理してから、企業の内部要因を評価する方が論理が通る。

### クロス SWOT（差別化のタネ）
4 象限を掛け合わせて戦略仮説を作る。**ES・面接で仮説をぶつけるのに使える**。

| | 機会 O | 脅威 T |
|---|---|---|
| **強み S** | **積極戦略**：強みで機会を取る（攻め） | **差別化戦略**：強みで脅威を回避 |
| **弱み W** | **改善戦略**：弱みを補強して機会を取る | **撤退 / 防衛戦略**：弱みと脅威の重なりを避ける |

例（NTT データ × AI ブームを想定）：
- S × O：SI のノウハウ × 生成 AI 需要急増 → 大企業向け生成 AI 導入支援を拡大できる
- W × T：レガシー依存 × クラウド化の波 → マルチクラウド対応力が急務

## 5. バリューチェーン分析

「どこで価値を生んでいるか」を分解。特にメーカー・SIer の研究に有効。

**製造業の例**：
研究開発 → 調達 → 製造 → 物流 → マーケ・営業 → 販売 → アフターサービス

どの工程が競争優位の源泉かを考える。
例：キーエンスは「直販営業」、トヨタは「調達＋製造」、村田製作所は「材料 R&D」。

## 6. VRIO 分析（経営資源の競争優位性）

強みが本当に持続的競争優位かを 4 つの軸で評価。

- **V**alue：経済的価値があるか
- **R**arity：希少性があるか
- **I**nimitability：模倣困難か
- **O**rganization：組織として活かせているか

**使い方**：「この会社の強みは何？」「それは本当に強みか？」を自分で検証。
面接で「弱みは？」と聞かれた時、V・R・I のうち何が欠けているかで語れる。

## フレームワークの使い分け（チートシート）

| 知りたいこと | 使うフレームワーク |
|---|---|
| 業界の大局観 | **PEST + 5 Forces** |
| 企業の競争優位 | **3C + VRIO** |
| 企業の戦略方向性 | **SWOT + クロス SWOT** |
| 事業の利益構造 | **バリューチェーン** |
| 事業ポートフォリオ | **PPM（プロダクト・ポートフォリオ・マトリクス）** |

## 就活で陥りがちな落とし穴

1. **フレームワークを埋めることが目的化する**
   → 埋めた後「だから何？」を必ず 1 行で書く

2. **情報収集に使うツールを間違える**
   → SWOT は解釈のフレーム。情報収集自体は PEST・3C・5 Forces で行う

3. **単一視点で完結させる**
   → PEST で外部を整理 → 3C で競合と自社を整理 → SWOT で統合、が推奨フロー

4. **数字で語らない**
   → 「強み：技術力」ではなく「強み：研究開発費 売上比 8%（業界平均 5%）、特許出願数 年 1,200 件」
